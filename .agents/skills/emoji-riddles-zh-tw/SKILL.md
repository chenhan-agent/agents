# 中文 Emoji 表意設計

這個 skill 的核心不是先自己發明一題猜謎，而是：

1. **使用者先給一個 target**
2. agent 根據這個 skill，把 target 轉成 4 個 emoji 的表意設計
3. 這組 emoji 要對 strong model 可解，但對 mini model 不夠直白

換句話說，這個 skill 是在教 agent：
**怎麼把任何指定 target 轉成夠有趣、夠機智、夠有層次的 emoji 表達。**

## 可用 target 範圍

可用 target 包含但不限於：

- 旅遊城市 / 景點 / 行程
- 食物 / 飲料
- 品牌 / 產品
- 電影 / 戲劇 / 動畫
- 名人 / 角色
- 成語 / 流行語
- 台灣文化梗 / 生活印象

旅遊只是其中一種 target，不是唯一用途。

## 什麼時候用

在這些情境使用這個 skill：

- 使用者已經給了明確 target，想把它轉成 4 個 emoji
- 想讓 emoji 設計偏繁中語感、偏台灣使用者直覺
- 想控制難度，讓 strong model 比較容易猜到，但 mini model 不要太容易秒中
- 想一次產生多組 emoji 設計，並附 target 對照與設計理由

## 核心原則

一組好的 emoji 表意設計應該同時滿足：

1. **4 個 emoji 就夠**
2. **至少有 2 層以上聯想**
3. **不是直接把 target 字面拆開**
4. **不是一眼百科題**
5. **target 揭曉後要讓人覺得合理，而且有記憶點**

## 難度目標

預設難度是：

- strong model：應能靠文化、語感、常識、諧音或意象串起來
- 弱模型 / mini model：應該只能猜到類型、範圍，或停在模糊聯想

真正的目標不是單純「做難」，而是做出 **對 full-tier 有解、對 mini-tier 不夠穩定** 的 emoji 設計。

如果 mini model 也很容易秒中，代表這組 emoji 還不夠好。

如果設計太直白，優先把其中 1 個 emoji 換成：

- 側面的文化線索
- 台灣人熟悉的生活印象
- 諧音、雙關或語意轉彎
- 不那麼 obvious、但事後會覺得很合理的聯想

## Separation-first 規則

為了拉開 full-tier 與 mini-tier，預設必須遵守：

1. **不要同時給出「target 俗稱」和「兩個以上經典配料 / 經典元素」**
2. **最多只允許 1 個直接指向 target 本體的 clue**
3. **至少 1 個 emoji 必須是側面文化線索，而不是食材、地標或外觀直譯**
4. **至少 1 個 emoji 必須承擔轉折或誤導功能**
5. **如果 4 個 emoji 全部都能在百科條目第一段找到，通常太直白**

對食物題尤其重要：

- 不要同時把「俗稱、主食材、兩個經典配料」全部給齊
- 若用了俗稱線索，其他線索就要更側面
- 若用了配料線索，至少有 1 個要轉成生活場景、文化印象或雙關

對旅遊題尤其重要：

- 不要直接丟最唯一的 landmark + 國家線索 + 經典食物
- 優先用城市氣質、交通印象、生活場景、旅人記憶點

對文化題尤其重要：

- 不要把角色外觀四件套直接排出來
- 至少留 1 個線索給「懂的人會會心一笑」的層次

## 適合拿來轉成 emoji 的 target

優先選這些適合做出層次的 target：

- 有鮮明印象但不只一個關鍵字的答案
- 台灣人相對熟悉的主題
- 可以同時從文化、語感、場景、食物、氣氛切入的 target

較推薦：

- 城市、地點、景點
- 夜市、飲料、食物
- 品牌、商店、產品
- 影視作品、角色
- 成語、俗語、迷因

## Target-driven 設計方法

優先從下面幾種線索中挑 3 到 4 種混搭：

- **文化符號**
- **食物 / 飲品**
- **氣候 / 場景 / 地理**
- **生活印象 / 使用情境**
- **角色特徵 / 劇情印象**
- **諧音 / 語感 / 雙關**
- **台灣人會秒有畫面的刻板印象**

最好讓 4 個 emoji 裡：

- 1 個偏核心
- 2 個偏輔助
- 1 個偏機智轉折

更具體地說：

- **核心 clue**：幫 full-tier 收斂範圍
- **輔助 clue**：讓 target 揭曉後合理
- **轉折 clue**：避免 mini-tier 只靠表層配對秒猜

## 設計流程

### 1. 先確認 target

先確認 target 是否有至少兩條以上聯想路徑。

如果 target 只能靠一個超直白特徵猜出來，通常不夠好。

### 2. 列出候選聯想

先寫出 5 到 8 個可能線索，再挑最有戲的 4 個。

不要直接選最 obvious 的 4 個。

### 3. 做層次分配

4 個 emoji 不要全部同類型。

盡量混搭：

- 1 個核心意象
- 1 個場景或文化線索
- 1 個生活或物件線索
- 1 個雙關、反差或側面聯想

如果 target 是台灣小吃、旅遊地點或熱門角色，優先避免這種過直白組合：

- 俗稱 + 本體 + 經典配料 + 經典配料
- 地標 + 地標 + 食物 + 國家
- 外觀 + 招牌道具 + 顏色 + 代表台詞

### 4. 自我檢查直白度

如果出現以下情況，通常太簡單：

- 直接把 target 拆成字面圖像
- 用唯一且毫無變形的招牌特徵
- 4 個 emoji 全都在講同一件事
- 看起來像百科快問快答而不是猜謎
- mini model 很可能只靠表面關鍵字就能直接猜中

### 5. 自我檢查可解性

如果 target 揭曉後有 2 個以上 emoji 很牽強，代表設計太飄，也要修。

## 避免的設計方式

避免：

- 直接拼字型拆解
- 只靠英文諧音，但中文使用者很難聯想
- 太冷門的在地梗，卻沒有其他支撐線索
- 為了難而難，導致 target 揭曉後也不覺得合理
- 一組裡 4 個 emoji 都在重複同一種提示
- 把 target 最經典的 3 到 4 個關鍵詞直接排成 emoji 清單
- 用 1 個俗稱 clue 就足以讓人秒答，其他 3 個只是在加強同一方向

## 推薦輸出格式

如果使用者沒有指定格式，預設輸出：

1. **target**
2. **emoji**：4 個 emoji
3. **設計理由**：每個 emoji 怎麼連到 target
4. **難度評語**：為什麼 strong model 較容易猜到，但 mini model 較容易卡住

如果使用者只要 emoji，不一定要主動展開完整解析；但在沒有特別限制時，最好附上設計理由。

## 風格要求

- 使用 **繁體中文**
- 口氣自然、有點機智，但不要油
- 可以帶台灣使用者熟悉的語感
- emoji 本身要精煉，不要偷塞文字提示

## 品質檢查

完成前檢查：

- 這組 emoji 是不是太直白？
- 這組 emoji 是不是其實不公平？
- 4 個 emoji 是否各自有功能？
- 公布 target 後，讀者會不會覺得「這連法很聰明」
- strong model 是否有足夠線索可推
- mini model 是否不太可能一次命中
- 如果把最強的 1 個 clue 拿掉，這組 emoji 是否仍然合理？
- 如果 mini model 只看表面關鍵字，是否會太容易直接鎖定 target？

## 預設輸出期待

使用這個 skill 時，預設產出：

1. 1 到多組 4-emoji 表意設計
2. 每組對應的 target
3. 每組的簡短設計理由
4. 每組的難度說明

## 範例方向

以下只是風格參考，不是固定模板：

- emoji 設計要像「看起來有點亂，但 target 揭曉很順」
- 不要像官方宣傳文案
- 不要像百科 facts 列表
- 旅遊題材可以做，但只是其中一種 target

## 邊界

這個 skill 的工作是 **把使用者指定的 target 轉成高品質、繁中、偏台灣語感的 4-emoji 表意設計**。

不是單純列資料，也不是做低難度兒童猜圖，更不是只能處理旅遊題。
