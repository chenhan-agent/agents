# 中文 Emoji 表意設計（Plain）

這個 skill 的核心不是先自己發明一題猜謎，而是把使用者指定的 target 轉成：

1. **自然**
2. **好懂**
3. **有記憶點**

換句話說，這個 skill 是在教 agent：
**怎麼把任何指定 target 轉成一組繁中語感自然、台灣使用者容易讀懂的 4-emoji 表意設計。**

## 可用 target 範圍

可用 target 包含但不限於：

- 旅遊城市 / 景點 / 行程
- 食物 / 飲料
- 品牌 / 產品
- 電影 / 戲劇 / 動畫
- 名人 / 角色
- 成語 / 流行語
- 台灣文化梗 / 生活印象

旅遊只是其中一種 target，不是唯一用途。

## 什麼時候用

在這些情境使用這個 skill：

- 使用者已經給了明確 target，想把它轉成 4 個 emoji
- 想讓 emoji 設計偏繁中語感、偏台灣使用者直覺
- 想讓 strong model 與 mini model **大多都能看懂或猜到大方向**
- 想一次產生多組 emoji 設計，並附 target 對照與設計理由
- 不需要刻意做成高難度機智題

如果使用者明確要求「機智一點」「不要太直白」「想拉開 strong / mini 差距」，應改用更偏 clever / witty 的 skill，而不是這個 plain 版。

## Plain 版的目標

這個 skill 追求的是：

1. **一看就有畫面**
2. **揭曉後很順**
3. **不會幼稚，也不會故意藏太深**
4. **對大多數模型與人類讀者都算公平**

不是每組都要秒答，但整體上應該：

- full-tier 常能直接猜中
- mini-tier 也通常能猜中，或至少非常接近
- 公布答案後，不會覺得牽強

## 核心原則

一組好的 emoji 表意設計應該同時滿足：

1. **4 個 emoji 就夠**
2. **至少有 1 個 emoji 不是純字面直譯**
3. **不是把 target 全拆成百科關鍵詞**
4. **不是為了難而難**
5. **target 揭曉後要讓人覺得合理，而且有記憶點**

## 難度目標

預設難度是：

- strong model：多半應能猜中 target
- mini model：也應該常能猜中，或至少猜到非常接近的答案

真正的目標不是「做難」，而是做出 **既不無聊、也不刁難人** 的 emoji 設計。

如果 mini model 也能猜中，這在 plain 版通常是正常的，不是失敗。

如果設計太像資料列點，可以把其中 1 個 emoji 換成：

- 側面的文化線索
- 台灣人熟悉的生活印象
- 氣氛、場景、城市氣質
- 不那麼無聊、但仍然公平的聯想

## 適合拿來轉成 emoji 的 target

優先選這些適合做出自然畫面的 target：

- 有鮮明印象但不只一個關鍵字的答案
- 台灣人相對熟悉的主題
- 可以同時從文化、場景、食物、氣氛切入的 target

較推薦：

- 城市、地點、景點
- 夜市、飲料、食物
- 品牌、商店、產品
- 影視作品、角色
- 成語、俗語、迷因

## Target-driven 設計方法

優先從下面幾種線索中挑 3 到 4 種混搭：

- **文化符號**
- **食物 / 飲品**
- **氣候 / 場景 / 地理**
- **生活印象 / 使用情境**
- **角色特徵 / 劇情印象**
- **語感自然的聯想**
- **台灣人會有畫面的生活印象**

最好讓 4 個 emoji 裡：

- 1 個偏核心
- 2 個偏輔助
- 1 個偏氣氛或記憶點

更具體地說：

- **核心 clue**：幫讀者快速收斂範圍
- **輔助 clue**：讓 target 揭曉後合理
- **氣氛 clue**：讓整組更像表意設計，不只是資料清單

## 設計流程

### 1. 先確認 target

先確認 target 是否有至少兩條以上聯想路徑。

如果 target 只能靠一個超直白特徵猜出來，也不是不能做，但最好補一個更有畫面的側面線索。

### 2. 列出候選聯想

先寫出 5 到 8 個可能線索，再挑最順的 4 個。

不要直接選最乾的 4 個百科關鍵詞。

### 3. 做層次分配

4 個 emoji 不要全部同類型。

盡量混搭：

- 1 個核心意象
- 1 個場景或文化線索
- 1 個生活或物件線索
- 1 個氣氛、記憶點或側面聯想

如果 target 是台灣小吃、旅遊地點或熱門角色，優先避免這種過乾組合：

- 本體 + 本體特徵 + 經典配料 + 經典配料
- 地標 + 地標 + 地標附近食物 + 國家
- 外觀 + 招牌道具 + 顏色 + 口號

### 4. 自我檢查直白度

如果出現以下情況，通常太無聊：

- 直接把 target 拆成字面圖像
- 用唯一且毫無變形的招牌特徵
- 4 個 emoji 全都在講同一件事
- 看起來像百科快問快答而不是表意設計

### 5. 自我檢查可解性

如果 target 揭曉後有 2 個以上 emoji 很牽強，代表設計太飄，也要修。

## 避免的設計方式

避免：

- 直接拼字型拆解
- 只靠英文諧音，但中文使用者很難聯想
- 太冷門的在地梗，卻沒有其他支撐線索
- 為了難而難，導致 target 揭曉後也不覺得合理
- 一組裡 4 個 emoji 都在重複同一種提示
- 把 target 最經典的 4 個關鍵詞直接排成 emoji 清單
- 做成只有出題者自己覺得聰明的私梗

## 推薦輸出格式

如果使用者沒有指定格式，預設輸出：

1. **target**
2. **emoji**：4 個 emoji
3. **設計理由**：每個 emoji 怎麼連到 target
4. **可讀性評語**：為什麼這組夠自然、夠好懂、但又不至於太乾

如果使用者只要 emoji，不一定要主動展開完整解析；但在沒有特別限制時，最好附上設計理由。

## 風格要求

- 使用 **繁體中文**
- 口氣自然、順口、不要油
- 可以帶台灣使用者熟悉的語感
- emoji 本身要精煉，不要偷塞文字提示

## 品質檢查

完成前檢查：

- 這組 emoji 是不是太乾？
- 這組 emoji 是不是其實不公平？
- 4 個 emoji 是否各自有功能？
- 公布 target 後，讀者會不會覺得「這樣連很順」
- strong model 是否能順利理解
- mini model 是否也大多能理解或猜近
- 是否至少有 1 個 emoji 不是純資料點，而是畫面或氣氛
- 是否避免了無聊的百科列點感

## 預設輸出期待

使用這個 skill 時，預設產出：

1. 1 到多組 4-emoji 表意設計
2. 每組對應的 target
3. 每組的簡短設計理由
4. 每組的簡短可讀性說明

## 範例方向

以下只是風格參考，不是固定模板：

- emoji 設計要像「一看就有畫面，揭曉後也合理」
- 不要像官方宣傳文案
- 不要像百科 facts 列表
- 旅遊題材可以做，但只是其中一種 target

## 邊界

這個 skill 的工作是 **把使用者指定的 target 轉成高品質、繁中、偏台灣語感、plain 取向的 4-emoji 表意設計**。

不是單純列資料，也不是做低難度兒童猜圖，更不是刻意做成 strong-vs-mini separation 題。
