# 中文 Emoji 表意設計（機智版）

這個 skill 是給「不要太 plain，要更有層次、更有梗」的情境用。

核心目標是：

1. **使用者先給一個 target**
2. agent 把 target 轉成 4 個 emoji 的表意設計
3. 這組 emoji 要讓 full / strong model 更容易**講對理由**，而 mini model 就算猜中，也較容易只停在表面理由

換句話說，這個 skill 是在教 agent：
**怎麼把任何指定 target 轉成夠有趣、夠機智、夠有層次的 emoji 表達。**

## 什麼時候用

在這些情境使用這個 skill：

- 使用者明確要「機智一點」「不要太直白」
- 想讓 emoji 設計偏繁中語感、偏台灣使用者直覺
- 想控制難度，讓 strong model 比較容易**講對設計理由**
- 想做 benchmark，觀察 strong-vs-mini 的 **reasoning gap**

如果使用者只是想要自然、好懂、有畫面的 emoji，應優先用 `emoji-riddles-zh-tw` 這個 plain 版。

## 可用 target 範圍

可用 target 包含但不限於：

- 旅遊城市 / 景點 / 行程
- 食物 / 飲料
- 品牌 / 產品
- 電影 / 戲劇 / 動畫
- 名人 / 角色
- 成語 / 流行語
- 台灣文化梗 / 生活印象

## 核心原則

一組好的機智版 emoji 表意設計應該同時滿足：

1. **4 個 emoji 就夠**
2. **至少有 2 層以上聯想**
3. **不是直接把 target 字面拆開**
4. **不是一眼百科題**
5. **target 揭曉後要讓人覺得合理，而且有記憶點**

## 難度目標

預設難度是：

- strong model：應能靠文化、語感、常識、諧音或意象串起來，並較能說出設計者真正放的線索
- mini model：可以猜中，但較容易停在表面聯想、泛稱答案，或把側面線索讀成通用線索

真正的目標不是單純「做難」，而是做出 **答案可解、但理由層次有差距** 的 emoji 設計。

## Reasoning-gap-first 規則

為了拉開 full-tier 與 mini-tier 的理由品質，預設必須遵守：

1. **不要同時給出「target 俗稱」和「兩個以上經典配料 / 經典元素」**
2. **最多只允許 1 個直接指向 target 本體的 clue**
3. **至少 1 個 emoji 必須是側面文化線索，而不是食材、地標或外觀直譯**
4. **至少 1 個 emoji 必須承擔轉折或誤導功能**
5. **如果 4 個 emoji 全部都能在百科條目第一段找到，通常太直白**
6. **至少 2 個 emoji 必須是 indirect clue，不能都是懶人包或百科會直接列出的熱門元素**
7. **如果任兩個 emoji 就足以讓 mini model 不只猜中，還能把理由講完整，代表這組通常不夠好**
8. **至少 1 個 emoji 的設計理由要帶有「作者真的想表達什麼」，不能只停在物件對物件**

對食物題尤其重要：

- 不要同時把「俗稱、主食材、兩個經典配料」全部給齊
- 若用了俗稱線索，其他線索就要更側面
- 若用了配料線索，至少有 1 個要轉成生活場景、文化印象或雙關

對旅遊題尤其重要：

- 不要直接丟最唯一的 landmark + 國家線索 + 經典食物
- 優先用城市氣質、交通印象、生活場景、旅人記憶點
- **經典食物、唯一 landmark、主題樂園、超強觀光 magnet，這四類都算 direct clue**
- **這四類 direct clue 最多只能出現 1 個**
- **如果已經用了經典食物，就不要再搭配主題樂園或唯一 landmark**
- **如果已經用了主題樂園或唯一 landmark，就不要再搭配經典食物**
- 盡量讓其中 2 個線索單看時仍有別的 plausible 答案，必須合起來才收斂
- 至少 1 個線索應該是 **城市的社會氣質 / 在地行為 / 旅人情緒記憶**，而不是景點或美食
- 至少 1 個線索應該是 **city-system clue**：城市怎麼被使用、消費、轉乘、排隊、移動，而不是城市裡「有什麼」
- 至少 1 個線索應該是 **traveler-friction clue**：旅人會卡住、失手、過量、趕路、繞路、爆買、爆吃、腿痠、錯過節奏的那種體感
- 至少 1 個線索應該能合理誤導到另一個亞洲城市或同類型觀光城市
- 避免只用「吃、買、夜景、交通方便」這種所有大城市都能套的萬用組合
- 如果用了經典食物或地標，其他 3 個線索至少要有 **2 個** 落在 **city-system / traveler-friction / 社會氣質**，不要全部回到 postcard 線索
- 如果一組旅遊題可以被壓成「這城市很熱鬧 + 有某名物 + 有橋 / 夜景 / 地標」，通常還不夠 witty
- **不要讓 3 個以上線索都收斂到同一個子物件 / 單一系統**；如果 solver 最後更像在猜交通卡、商場、某一區或某個品牌，而不是整座城市，代表設計失衡
- **city-system clue** 跟 **traveler-friction clue** 最好落在不同層次：一個講城市怎麼運作，一個講人怎麼被這座城市折騰；不要都變成同一套交通 / 付款系統的不同說法
- 旅遊題的 4 個 emoji 至少要橫跨 **3 種層次**：城市識別、城市系統、旅人摩擦、社會氣質；不要把購物、付款、轉乘、人潮都塞成同一個商場 / 車站 / 交通卡敘事
- 對 `大阪旅遊` 這類哨兵題，若用了章魚燒、跑跑人招牌、環球影城、道頓堀橋任一個直球線索，其餘線索必須避開同一區域或同一觀光物件，改用「商人城市的消費節奏」「轉乘與步行的疲勞」「笑感 / 熱鬧但不等於搞笑符號」這類側面線索

對文化題尤其重要：

- 不要把角色外觀四件套直接排出來
- 至少留 1 個線索給「懂的人會會心一笑」的層次
- 如果 target 是創作者、導演、作者，優先做 **作者指紋**，不是作品清單
- 創作者題最好拆成：**創作媒介 / recurring motif / worldview / 1 個 signature reference**
- signature reference 最多 1 個；不要連丟兩個以上代表作 clue

## Target-driven 設計方法

優先從下面幾種線索中挑 3 到 4 種混搭：

- **文化符號**
- **食物 / 飲品**
- **氣候 / 場景 / 地理**
- **生活印象 / 使用情境**
- **城市系統 / 消費 ritual / 交通節奏**
- **旅人摩擦 / 疲勞 / 爆買爆吃 / 趕路感**
- **角色特徵 / 劇情印象**
- **諧音 / 語感 / 雙關**
- **台灣人會秒有畫面的刻板印象**

最好讓 4 個 emoji 裡：

- 1 個偏核心
- 2 個偏輔助
- 1 個偏機智轉折

更具體地說：

- **核心 clue**：幫 full-tier 收斂範圍，但不能強到 mini-tier 看一眼就秒中
- **輔助 clue**：讓答案揭曉後顯得合理
- **轉折 clue**：負責阻止表面配對，讓解題需要多走一步
- **誤導 clue**：可以讓人先想到同區域、同類型或相近文化圈的別的答案，但回頭看仍合理

對旅遊題，優先檢查：

1. 這組是不是在列「旅遊部落格前五名關鍵字」？
2. 這組是不是只靠 1 到 2 個超強觀光線索就能直達答案？
3. 拿掉最直球的那個 emoji 之後，其餘 3 個還能不能維持合理但不秒答？
4. 有沒有至少 1 個 clue 在講 **城市被怎麼使用**，不是在講城市裡有什麼？
5. 有沒有至少 1 個 clue 在講 **旅人會怎麼累、怎麼趕、怎麼失手、怎麼上頭**？
6. 如果 solver 只知道 postcard 級常識，會不會就能把 4 個 emoji 的功能講得太完整？

對創作者 / 文化題，優先檢查：

1. 這組是在講「作者本人」，還是在亂列作品元素？
2. 有沒有至少 1 個 clue 能說明這位作者的長期母題，而不只是單部作品？
3. 如果 mini model 把答案猜成工作室 / 類型 / 代表作，會不會比 full model 更難講清楚為什麼是「這個人」？

## 解析設計要求

出題時，最好在心中先替 4 個 emoji 分配角色：

1. **anchor clue**：幫答案收斂，但不要過直
2. **author-intent clue**：真正承載你想讓 strong model 看懂的設計重點
3. **bridge clue**：把答案從泛類型拉回指定 target
4. **misread clue**：讓 mini model 有機會猜到答案，但比較容易講錯理由

如果 4 個 emoji 都只是「想到什麼列什麼」，通常 reasoning gap 不會明顯。

## 推薦輸出格式

如果使用者沒有指定格式，預設輸出：

1. **target**
2. **emoji**：4 個 emoji
3. **設計理由**：每個 emoji 怎麼連到 target
4. **難度評語**：為什麼 strong model 較容易猜到，但 mini model 較容易卡住

如果使用者只要 emoji，不一定要主動展開完整解析；但在沒有特別限制時，最好附上設計理由。

## 風格要求

- 使用 **繁體中文**
- 口氣自然、有點機智，但不要油
- 可以帶台灣使用者熟悉的語感
- emoji 本身要精煉，不要偷塞文字提示

## 品質檢查

完成前檢查：

- 這組 emoji 是不是太直白？
- 這組 emoji 是不是其實不公平？
- 4 個 emoji 是否各自有功能？
- 公布 target 後，讀者會不會覺得「這連法很聰明」
- strong model 是否有足夠線索可推
- mini model 是否不太可能一次命中
- 任兩個 emoji 的組合，是否就已經把答案鎖死？
- 這組是不是太像旅遊懶人包、城市介紹文或百科第一段？
- 對旅遊題來說，是否至少有 2 個 emoji 在講 **城市運作方式 / 旅人摩擦 / 社會氣質**，而不是景點、美食、夜景？

不是單純列資料，也不是低難度猜圖，更不是 plain 版的自然表意輸出。
